# AI Agents Assessment — Intermediate

## Purpose

Evaluate practical AI agent engineering knowledge including architecture, planning, tool usage, memory, workflows, and production considerations.

This assessment validates the ability to design reliable agent-based systems.

---

## Assessment Scope

Topics covered:

- Agent architecture
- Planning and reasoning workflows
- Tool integration
- Memory strategies
- Agent execution lifecycle
- Observability
- Error handling
- Production considerations

---

## Assessment Format

Three levels:

### Level 1 — Concept Understanding

Validates intermediate AI agent concepts.

### Level 2 — Engineering Scenarios

Validates practical agent design decisions.

### Level 3 — Senior Engineer Decisions

Advanced architecture assessment is covered separately.

# Level 2 — Engineering Scenarios

## Q4. A deployment troubleshooting agent receives a failure report.

Design the workflow.

Expected thinking:

- Understand objective
- Collect context
- Retrieve logs and metrics
- Analyze information
- Execute available tools
- Validate recommendation

---

## Q5. An agent repeatedly performs incorrect actions.

Investigation areas:

- Prompt instructions
- Planning logic
- Tool outputs
- Context quality
- Validation mechanisms

---

## Q6. An agent requires access to infrastructure APIs.

What controls are required?

Expected thinking:

- Least privilege access
- Tool authorization
- Audit logging
- Human approval for sensitive operations
- Failure handling

---

# Level 3 — Senior Engineer Decisions

## Q7. Design an engineering assistant agent for production support.

Consider:

- User interaction
- Planning engine
- Knowledge retrieval
- Tool execution
- Security boundaries
- Observability
- Evaluation

---

## Q8. When should an AI agent not be used?

Expected thinking:

- Simple deterministic workflow
- Strictly predictable automation
- High-risk action without approval
- No clear business value

---

## Q9. Production issue:

```
Agent Response: Incorrect
Tools: Available
Model: Available
User Context: Incomplete
```

Investigation areas:

- Missing context
- Retrieval quality
- Tool results
- Prompt design
- Validation process
