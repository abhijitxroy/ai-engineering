# AI Agents Assessment — Beginner

## Purpose

Evaluate foundational understanding of AI agents, their components, workflows, and engineering use cases.

This assessment validates basic knowledge required before designing and building agent-based systems.

---

## Assessment Scope

Topics covered:

- AI agent fundamentals
- Traditional automation vs agents
- Agent components
- Planning concepts
- Tool usage
- Memory concepts
- Agent workflows
- Basic engineering applications

---

## Assessment Format

Three levels:

### Level 1 — Concept Understanding

Validates AI agent fundamentals.

### Level 2 — Engineering Scenarios

Validates practical understanding of agent behavior.

### Level 3 — Senior Engineer Decisions

Advanced architecture assessment is covered separately.

# Level 2 — Engineering Scenarios

## Q4. A deployment troubleshooting agent receives a failure report.

Possible agent workflow:

Expected thinking:

- Collect context
- Analyze logs
- Retrieve relevant information
- Select tools
- Execute actions
- Validate results

---

## Q5. An AI agent gives incorrect recommendations.

What should be investigated first?

Expected thinking:

- Input context quality
- Tool results
- Prompt instructions
- Validation process
- Available knowledge

---

## Q6. A team wants an agent to access production systems.

What controls are required?

Expected thinking:

- Permission boundaries
- Tool restrictions
- Audit logging
- Human approval for critical actions

---

# Level 3 — Engineering Thinking

## Q7. Design a simple engineering assistant agent.

Consider:

- User request handling
- Context collection
- Planning
- Tool usage
- Response validation

---

## Q8. When should a traditional workflow be preferred over an AI agent?

Expected thinking:

- Fully deterministic processes
- Strict compliance requirements
- Predictable execution paths

---

## Q9. Why is evaluation important for AI agents?

Expected thinking:

- Measure task success
- Detect failures
- Improve reliability
- Prevent unsafe actions
