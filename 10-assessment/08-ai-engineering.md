# AI Engineering Assessment

## Purpose

Evaluate modern AI engineering knowledge including AI application development, LLM integration, AI workflows, and production AI system thinking.

This assessment validates the ability to build reliable AI-powered software systems using engineering principles.

---

## Assessment Scope

Topics covered:

- AI engineering fundamentals
- AI application architecture
- LLM integration
- Prompt engineering
- RAG systems
- AI evaluation
- AI safety
- Production AI operations
- Cost and performance optimization

Three levels:

### Level 1 — Concept Understanding

Validates AI engineering fundamentals.

### Level 2 — Engineering Scenarios

Validates practical AI application decisions.

# Level 2 — Engineering Scenarios

## Q4. An AI assistant provides incorrect answers from company documents.

What should be investigated?

Expected thinking:

- Retrieval quality
- Document quality
- Chunking strategy
- Embedding model
- Prompt design
- Evaluation metrics

---

## Q5. An AI application has high latency and expensive API usage.

Possible improvements:

- Model selection
- Prompt optimization
- Caching
- Response streaming
- Request batching

Expected thinking:

Measure cost and latency before optimization.

---

## Q6. A team builds an AI feature but cannot verify response quality.

What should be added?

A. More UI features

B. Evaluation framework and quality metrics

C. More infrastructure only

# Level 3 — Senior Engineer Decisions

## Q7. Design an enterprise AI assistant.

Consider:

- User authentication
- Data privacy
- Knowledge retrieval
- Model selection
- Evaluation
- Monitoring
- Security controls

---

## Q8. A company wants to introduce AI agents into engineering workflows.

What should be evaluated?

Expected thinking:

- Agent responsibilities
- Tool permissions
- Human approval points
- Security boundaries
- Observability
- Failure handling

---

## Q9. Production AI issue:

```
Application: Running
Model API: Available
Response quality: Decreasing
User complaints: Increasing
```

Investigation areas:

- Prompt changes
- Data changes
- Retrieval failures
- Model version changes
- Evaluation results
