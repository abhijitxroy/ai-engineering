# Logs

Purpose:

Capture events generated by systems.

Examples:

Application logs

Deployment logs

Infrastructure logs

Example:

Deployment failed.

Log:

Connection timeout to dependency.

Benefits:

- Failure investigation
- Event visibility
- Root cause support

Challenges:

- High volume
- Signal extraction difficulty

Questions:

What happened?

---

# Metrics

Purpose:

Track numerical operational information.

Examples:

CPU utilization

Memory utilization

Latency

Error rate

Request throughput

Example:

Latency increased.

Error rate increased.

Potential operational issue detected.

Benefits:

- Trend visibility
- Capacity understanding
- Operational monitoring

Questions:

System healthy?

---

# Tracing

Purpose:

Track request lifecycle across dependencies.

Workflow:

User Request

↓

Gateway

↓

Service A

↓

Service B

↓

Database

↓

Response

Example:

Latency increase observed.

Tracing identifies:

Service B dependency delay.

Benefits:

- Dependency visibility
- Bottleneck identification
- Distributed troubleshooting

Questions:

Where issue occurred?
